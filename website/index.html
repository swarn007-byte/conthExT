System Architecture

The Ingestion Pipeline

Watch how Claude-style messages move through the local backend, land inside Obsidian, and become queryable vault memory.

[conthExT Flow] Frontend or Claude sends ContextPayload → FastAPI queues processing → Obsidian Threads/Concepts store memory → Vault graph grounds the next answer.
Product Capabilities

A frontend for the memory you already own.

The UI sits on top of your local backend and vault, turning raw markdown threads into a usable agent workspace.

Cross-Tool Persistence

Captures user and agent messages through a simple API and persists them as readable Obsidian markdown.

Temporal Thread Clustering

Keeps every session inside a named thread so old decisions, bugs, and explanations remain easy to retrieve.

Entity Synapse Extraction

Turns thread notes, wiki-links, and concepts into graph nodes that the sandbox can search and focus visually.

Citation-Backed Grounding

Every sandbox answer points back to the matching vault note instead of pretending memory appeared from nowhere.

Model-Agnostic Interface

Built around REST today and ready for Claude/Anthropic tool-calling or MCP-backed vault operations next.

Zero Surprise Costs

Runs locally against filesystem markdown. No external vector store is required for the current sandbox loop.

Comparison Matrix

Why vault-native memory is the right first layer.
